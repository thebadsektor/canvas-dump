What is a Database?

  • A database is an organized collection of related data that represents some aspect of the real world and is stored and accessed electronically.
  • Databases are designed to efficiently manage large amounts of information by providing a systematic way to store, retrieve, and manipulate data.
  • The definition has evolved from any number of data items in a list to a vault of structured data, or records, accessible via a computer system.
    • A record is the representation of a single item, such as a customer, and contains various attributes like names, phone numbers, and addresses.

Key Concepts: Data and Metadata

A database contains more than just data; it also includes metadata.

  • Metadata is data that defines the structure of the information within the database.
  • Because a database contains descriptions of its own structure, it is considered self-describing.
  • Knowing how data is organized allows you to access, manage, maintain, and modify it.
  • Metadata is stored in a data dictionary, which describes the components of a database, such as tables, rows, and columns.
    • In contrast, flat-file systems do not contain metadata.

Classifying Databases

Databases can be classified by their size, the number of people who need access, and the technical equipment required.

  1. Personal Databases: These are the smallest and are typically stored on a user's computer with a basic data structure.
  2. Group Databases: More complex than personal databases, they are intended for use by a department or team. They contain more data and must be accessible from multiple devices simultaneously.
  3. Enterprise Databases: These are large, complex databases that require the most reliable equipment to be safely stored and maintained.

The Fundamental Structure: Tables

  • In a database, tables are the fundamental structures that organize and store data with precision.
  • A table is defined as a collection of related data organized in the form of rows and columns.
  • This structure enhances the performance of information control and retrieval.
  • A well-structured table is a core component of relational database design, providing a logical framework for storing and retrieving data.

Elements of a Table (Part 1)

  • Columns (Fields or Attributes): This is the vertical information in a table. Each column has a unique name (e.g., ID, Name, Age) and allows for sorting and filtering data.
  • Rows (Tuples or Records): This is the horizontal data for each entry. A single row holds the complete data for a specific item, such as all information for one student.
  • Data Items (Column Values): This is the specific piece of information stored in a cell where a row and column intersect.

Elements of a Table (Part 2)

  • Data Types: Every column is assigned a specific data type (e.g., int, varchar, date) that defines the kind of data it can hold. This ensures the data follows a particular format and structure.
  • Primary Key: This is a unique identifier for each row in a table. A table can only have one primary key, it cannot be null, and it is often the first column. For example, a student's ID can be a primary key.
  • Foreign Key: This is a column (or set of columns) that refers to the primary key of another table, establishing a relationship between the two tables. A table can have multiple foreign keys, and they can be null.

Lesson Summary

  • A database is a self-describing, organized collection of related data.
  • Databases can be categorized as personal, group, or enterprise based on size, users, and technical needs.
  • The table is the primary structure for organizing data in rows and columns.
  • Tables are made of essential elements like columns, rows, data types, and keys (Primary and Foreign), which work together to ensure data integrity and effective manipulation.
