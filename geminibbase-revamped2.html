Be Specific & Explicit

Clearly state your target. Instead of "find the price," use "extract the text from the element with the selector '.price-tag'."

One Action Per Step

Break down complex tasks into a sequence of simple commands. Go to a URL, then click a button, then type text. This improves reliability.

üîé

Use Stable Selectors

Reference elements by stable IDs or data attributes when possible (e.g., `[data-testid="submit-button"]`). Relying on vague text like "Click the 'Next' button" can be brittle.

Describe Loops Clearly

When you need to repeat actions, define the scope. For example, "For each element matching '.product-card', extract the title and price."

Director Recipes for Profit

Recipe 1: E-commerce Price Monitoring

Profit Strategy: Price comparison service, drop-shipping deal finder, or competitor price tracking for a client.

The Director Prompt:

1. Go to "https://www.amazon.com/dp/B08P2H5LW7".
2. Wait for the element with selector "#corePrice_feature_div" to be visible.
3. Extract the text from the element with selector "span.a-price-whole" and the element with selector "span.a-price-fraction".
4. Return the extracted texts.

Step-by-Step Breakdown:

  • `Go to...`: Navigates the browser to a specific product page.
  • `Wait for...`: Crucial for dynamic pages. It prevents the script from failing before the price information loads.
  • `Extract the text...`: Precisely targets the dollars and cents of the price using their specific CSS selectors.
  • `Return...`: Makes the extracted data available as the output of the automation.

Recipe 4: Lead Generation from a Directory

Profit Strategy: Build and sell targeted lead lists for B2B sales teams or marketing agencies.

The Director Prompt:

1. Go to "https://www.yelp.com/search?find_desc=plumbers&find_loc=San+Francisco%2C+CA".
2. Wait for 5 seconds for the page to load completely.
3. For each element that matches the CSS selector "[data-testid='biz-name']", do the following:
   a. Extract the text of the element.
   b. Extract the 'href' attribute of the element.
4. Go to the next page by clicking the element with selector "[aria-label='Next']".
5. Repeat steps 3-4 three times.
6. Return the collected data.

Step-by-Step Breakdown:

  • `Go to...`: Starts at a pre-defined search result page on Yelp.
  • `For each element...`: This is the core looping instruction. It finds all business name links and performs nested actions.
  • `Extract the 'href' attribute`: Shows how to get more than just visible text, like the URL to the business's Yelp page.
  • `Go to the next page...`: Demonstrates pagination, a key skill for scraping lists.
  • `Repeat steps...`: Controls how many pages the automation will process.

Recipe 5: Real-Time Inventory Check

Profit Strategy: Automation for retail arbitrage or dropshipping, alerting you to low-stock or newly restocked products.

The Director Prompt:

1. Go to "https://www.bestbuy.com/site/apple-airpods-pro-2nd-generation".
2. Wait for the button with the text "Add to Cart" to be visible.
3. Check if the button is enabled.
4. If the button is enabled, return "In Stock".
5. If the button is not enabled, return "Out of Stock".

Step-by-Step Breakdown:

  • `Go to...`: Navigates to a specific product page.
  • `Wait for...`: Ensures the page has fully loaded the critical "Add to Cart" button, which can be dynamically rendered.
  • `Check if...`: A conditional command that checks the interactive state of an element, which is a powerful way to determine inventory status.
  • `If... return...`: The final conditional logic that provides a clear, actionable output based on the button's state.

Recipe 7: Aggregating Real Estate Listings

Profit Strategy: Create a niche real estate aggregation service that alerts users to properties meeting very specific criteria.

The Director Prompt:

1. Set a variable 'listings' to an empty array.
2. Go to "https://www.zillow.com/homes/for_rent/New-York,-NY_rb/".
3. Wait for the property cards to load by waiting for an element with selector "article.property-card".
4. Get all elements matching "article.property-card" and store them in a variable 'cards'.
5. For each 'card' in 'cards':
   a. Inside the 'card', find the element with selector "[data-test='property-card-price']" and extract its text into a 'price' variable.
   b. Inside the 'card', find the element with selector "[data-test='property-card-addr']" and extract its text into an 'address' variable.
   c. Inside the 'card', find the 'a' element and get its 'href' attribute into a 'link' variable.
   d. Add an object with 'price', 'address', and 'link' to the 'listings' array.
6. Check if an element with selector "a[title='Next page']" exists. If it does, click it and go back to step 3.
7. Return the 'listings' array.

Step-by-Step Breakdown:

  • `Set a variable...`: Initializes an array to store the results.
  • `For each 'card' in 'cards'`: Iterates through each property listing.
  • `Inside the 'card', find...`: This demonstrates "scoped" searching, which is vital for accurate data extraction from lists.
  • `Check if...`: The conditional logic to handle pagination, ensuring the automation scrapes the entire result set.

Recipe 8: Dynamic Form Filling & Submission

Profit Strategy: Automate complex grant applications or permit filings for a fee, saving clients massive amounts of time.

The Director Prompt:

1. Go to "https://example.com/complex-application".
2. Type "John Doe" into the input field with the name "applicantName".
3. Type "555-123-4567" into the input field with the name "phoneNumber".
4. Click the radio button with the value "individual".
5. Click the button with the text "Next Step".
6. Wait for the page to navigate to the next step.
7. Repeat steps 2-6 for each section of the form until the "Submit" button is visible.
8. Click the button with the text "Submit".
9. Return the text "Form submitted successfully."

Step-by-Step Breakdown:

  • `Go to...`: Navigates to the first page of the form.
  • `Type...`: Directs the automation to input specific text into form fields. Using the `name` attribute is a highly stable approach.
  • `Click...`: Interacts with specific UI elements like buttons and radio buttons.
  • `Repeat...`: This advanced command handles multi-step forms by repeating a sequence of actions until a specific condition (the final submit button) is met.

Jumbo Edge Case Recipe: Customer Service Chatbot Training Data

Profit Strategy: Sell highly-structured, cleaned data to companies for training their internal AI chatbots and knowledge bases.

The Director Prompt:

1. Set a variable 'qa_data' to an empty array.
2. Go to "https://support.google.com/chromebook/?hl=en#topic=3399709".
3. Wait for all elements with the class ".support-thread-card" to be visible.
4. For each element that matches the selector ".support-thread-card":
   a. Extract the text of the element with the selector ".thread-title" and store it in a variable 'question'.
   b. Click the link with the selector ".thread-title" to navigate to the thread page.
   c. Wait for the element with the selector ".thread-answer-text" to be visible.
   d. Extract the text of the element with the selector ".thread-answer-text" and store it in a variable 'answer'.
   e. Add a new object {'question': question, 'answer': answer} to the 'qa_data' variable.
   f. Go back to the previous page.
5. Go to the next page by clicking the element with the selector "[aria-label='Next page']".
6. Repeat steps 4-5 five times.
7. Return the 'qa_data' array.

Step-by-Step Breakdown:

  • `Set a variable...`: Initializes an array to store structured Q&A objects.
  • `For each element...`: The outer loop that handles each question card on the list page.
  • `Click... then Go back...`: This is the critical, complex part. The automation navigates into each individual question to get its specific answer and then returns to the list page to continue the loop.
  • `Add a new object...`: Creates a structured JSON object for each Q&A pair, ready for use as training data.
  • `Repeat...`: This handles both the inner loop (processing each item on a page) and the outer loop (moving to the next page), demonstrating a highly advanced, nested automation.
